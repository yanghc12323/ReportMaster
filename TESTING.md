## 4. 工作流主链路（v0.6 核心）

### 4.1 输入校验

- [ ] 主题为空时点击“开始生成”会提示“请输入主题”
- [ ] 未配置 API Key 时会提示缺失角色并跳转到配置页

### 4.2 五角色完整执行

- [ ] 启动后可看到如下顺序的执行痕迹：
  - [ ] 结构规划者
  - [ ] 调研者
  - [ ] 主笔人
  - [ ] 编辑
  - [ ] 审稿人

### 4.3 审稿决策机制（直到接收）

- [ ] 审稿结论仅落在：大修 / 小修 / 接收（或被系统归一化）
- [ ] 若出现“拒稿/不接收”语义，系统按“大修”继续迭代
- [ ] 大修时触发下一轮完整主链路
- [ ] 小修时触发“编辑小修 → 再次送审”
- [ ] 仅在“接收”时触发 workflow_complete

### 4.4 模式行为（v0.6）

- [ ] quick / standard / full 均可正常启动
- [ ] 三种模式当前均采用“直到审稿接收”的同一策略
- [ ] `workflow_start` 事件中 `max_iterations` 为 `unlimited`
